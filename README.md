# 🧊 스마트 냉장고 — AI 기반 식품 관리 & 레시피 추천 플랫폼

> 냉장고 속 식재료를 등록하고, AI가 맞춤 레시피를 추천해주는 웹 서비스

---

## 📌 프로젝트 개요

사용자가 냉장고에 있는 재료들을 플랫폼에 등록하고 관리하면,  
AI가 자동으로 남은 재료를 중심으로 레시피를 맞춤 추천하고,  
유통기한이 얼마 안 남은 식품을 알려주는 **AI 기반 식품 관리 & 레시피 추천 플랫폼**입니다.

단순한 식재료 관리를 넘어 **냉장고 관리 → 레시피 보고 요리하기 → 부족한 재료 장보기**의 전체 과정을 하나의 흐름으로 연결하는 것을 목표로 합니다.

---

## ✨ 주요 기능

| 기능 | 설명 |
|------|------|
| 🥗 식재료 등록 & 관리 | 식품명, 수량, 보관 위치, 유통기한을 입력해 냉장고를 관리 |
| 🔍 검색 & 필터 | 식품명 검색, 냉장/냉동/실온 필터링 |
| ⏰ 유통기한 관리 | 만료 여부에 따라 색상으로 자동 분류 (빨강/노랑/초록) |
| ✨ AI 레시피 추천 | 현재 재료로 만들 수 있는 레시피 + 재료를 조금만 추가하면 되는 레시피 추천 |
| 👍 취향 반영 | 사용자가 선택한 레시피 이력을 저장해 점점 개인화된 추천 제공 |

## 🖥️ 화면 구성

### 메인 화면
- 냉장고 현황 요약 (전체 식품 수 / 임박 / 만료)
- 보관 위치별 현황 (냉장 / 냉동 / 실온)
- 식재료 목록 카드 (유통기한 임박순 정렬)

### 식품 추가 화면
- 식품명, 수량, 보관 위치, 유통기한 입력 후 추가

### AI 레시피 추천 화면
- **지금 바로 만들 수 있어요** — 현재 보유 재료만으로 완성 가능한 레시피
- **재료 조금만 사면 만들 수 있어요** — 1~2가지만 추가 구매하면 되는 레시피
- 각 레시피에 사용 재료, 부족한 재료, 조리법 포함

## 🗄️ 데이터베이스 설계 (ERD)

### 핵심 엔티티

#### 👤 USER (사용자 계정 정보)
> 서비스를 이용하는 주체의 기본 인증 정보 관리

| 컬럼 | 타입 | 설명 |
|------|------|------|
| user_id | INT (PK) | 고유 식별키 |
| username | STRING | 사용자명 |
| email | STRING | 로그인을 위한 이메일 |
| password | STRING | 비밀번호 |

---

#### 🧊 FRIDGE_ITEM (사용자별 보유 식재료)
> 냉장고 상태 트래킹을 담당하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| item_id | INT (PK) | 고유 식별키 |
| user_id | INT (FK) | 어떤 사용자의 냉장고인지 식별 |
| name | STRING | 식품명 |
| quantity | STRING | 식재료 수량 (예: 3개, 500ml) |
| storage_location | STRING | 보관 위치 (냉장 / 냉동 / 실온) |
| registered_date | DATE | 등록일 |
| expiration_date | DATE | 유통기한 |

---

#### 🍽️ USER_RECIPE (사용자 맞춤 취향 이력)
> 사용자가 선호하여 선택했던 요리의 메타데이터 관리

| 컬럼 | 타입 | 설명 |
|------|------|------|
| recipe_id | INT (PK) | 고유 식별키 |
| user_id | INT (FK) | 어떤 사용자가 선택했는지 |
| recipe_name | STRING | 레시피 이름 (요리명) |
| category | STRING | 카테고리 (한식 / 일식 / 중식 등) |

---

### 엔티티 관계

```
USER ──< FRIDGE_ITEM   (1:N) — 한 사용자는 여러 식재료를 등록할 수 있다
USER ──< USER_RECIPE   (1:N) — 한 사용자는 여러 레시피를 선택할 수 있다
```

## ⚙️ 시스템 아키텍처

```
[사용자]
   ↕ 클릭/입력
[React (프론트엔드)]
   ↕ HTTP 요청 (Axios / Fetch)
[Spring Boot (백엔드)]
   ├─ Filter → DispatcherServlet → Interceptor (보안/인증)
   ├─ Controller → Service → Repository
   └─ DTO → JSON 응답
[Database]
```

### 요청 흐름
1. **요청 발생 (React)** — 사용자 버튼 클릭 → HTTP 요청
2. **요청 수신 (Spring Boot)** — 필터 → 인증 검증 → 컨트롤러 전달
3. **로직 처리** — Service 호출 → Repository로 DB 연동 → DTO 가공
4. **응답 반환** — JSON으로 자동 변환 → 클라이언트 전달
5. **화면 업데이트 (React)** — State 업데이트 → 리렌더링

## 👤 사용자 시나리오

1. **식재료 등록** — 메인화면 → "식품 추가" 버튼 → 정보 입력 → 냉장고 목록에 자동 반영
2. **유통기한 확인** — 접속 시 임박 식품 빨강/노랑으로 강조 표시
3. **AI 레시피 추천** — "AI 레시피 추천받기" 버튼 → 현재 재료 기반 레시피 목록 확인 → 레시피 선택
4. **취향 데이터 축적** — 선택한 레시피가 USER_RECIPE에 저장 → 추천 개인화
